# Task Queue Service

Сервис очереди задач на обработку файлов. Принимает задачи, ставит их в очередь, обрабатывает асинхронно и сохраняет результат.

## Технологии

Kotlin, Spring Boot, Coroutines, H2/PostgreSQL, Spring Data JPA, Liquibase, Swagger, JUnit, MockK

## Запуск

```bash
./gradlew bootRun
# или
docker-compose up --build
```

Приложение доступно на `http://localhost:8080`.

Swagger: `http://localhost:8080/swagger-ui.html`

H2 Console: `http://localhost:8080/h2-console` (JDBC URL: `jdbc:h2:mem:taskqueue`, user: `sa`, пароль пустой)

## Тесты

```bash
./gradlew test
```
Реализованные функции
Обязательные

    Создание, просмотр, отмена задач

    Список задач с фильтрацией по статусу, типу, дате, пути, приоритету

    Пагинация

    Асинхронная обработка через Kotlin Coroutines

    Ограничение одновременных задач через Semaphore (настраивается в конфиге)

    Корректные переходы статусов (PENDING → PROCESSING → COMPLETED/FAILED)

    Сохранение ошибок обработки

    Валидация входных данных

Дополнительные

    Retry для упавших задач (3 попытки, настраивается)

    Scheduler для поднятия зависших задач (каждые 5 минут)

    История смены статусов (отдельная таблица task_events)

    Приоритет задач

    Метрики выполнения (время обработки, счётчики статусов, активные воркеры)

    Стратегии обработки разных типов файлов (CSV, изображения, PDF)
